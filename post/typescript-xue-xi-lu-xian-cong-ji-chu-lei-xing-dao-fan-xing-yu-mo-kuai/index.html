TypeScript 学习路线：从基础类型到泛型、模块与类型声明

技术 · 2023-07-31 · 10 分钟阅读
TypeScript

TypeScript 是 JavaScript 的超集。它不会取代 JavaScript，而是在 JavaScript 之上增加静态类型检查、编辑器提示和更可靠的重构能力，最终仍会被编译成 JavaScript 运行。

面对 TypeScript 庞大的类型系统，初学者很容易陷入“知识点都看过，项目里却不会用”的困境。本文以学习路线图为线索，从环境搭建、基础类型一路讲到接口、泛型、模块和声明文件，帮助你建立一套可以落地到项目中的知识框架。

一、为什么学习 TypeScript

JavaScript 的灵活性提高了开发效率，也让一些错误只能在运行时暴露：

function formatUser(user) {
  return user.name.toUpperCase()
}

formatUser({ name: null }) // 运行到这里才报错

TypeScript 可以提前描述参数的形状，让错误在编码或编译阶段被发现：

interface User {
  name: string
}

function formatUser(user: User): string {
  return user.name.toUpperCase()
}

它带来的主要收益包括：更早发现错误、更准确的代码补全、更安全的重构，以及更清晰的函数和数据结构文档。

二、安装与运行原理

先准备 Node.js 环境，再在项目中安装 TypeScript：

npm install --save-dev typescript
npx tsc --init
npx tsc --version

假设有 hello.ts：

const message: string = "Hello TypeScript"
console.log(message)

执行 npx tsc hello.ts 后会生成 JavaScript 文件。其核心流程可以概括为：

TypeScript 源码 → 类型检查 → JavaScript 源码 → 浏览器或 Node.js 执行

类型信息主要服务于开发和编译阶段，通常不会保留到最终的 JavaScript 中。

三、基础类型与类型推断

1. 常见基础类型

const count: number = 10
const title: string = "TypeScript"
const enabled: boolean = true
const bigValue: bigint = 100n
const uniqueKey: symbol = Symbol("key")

let empty: null = null
let missing: undefined = undefined

数组有两种常见写法：

const scores: number[] = [90, 95, 100]
const names: Array<string> = ["Alice", "Bob"]

2. 类型推断

能让 TypeScript 推断时，不必为每个变量重复标注类型：

const language = "TypeScript" // 推断为 string
const version = 5              // 推断为 number

类型标注更适合用在函数边界、公共 API 和结构复杂的数据上。

3. any、unknown 与 never

any 会绕过类型检查，应谨慎使用。面对类型暂时未知的数据，优先选择 unknown：

function printValue(value: unknown): void {
  if (typeof value === "string") {
    console.log(value.toUpperCase())
  }
}

never 表示不可能出现的值，常用于永远不会正常结束的函数或穷尽检查：

function fail(message: string): never {
  throw new Error(message)
}

四、变量、条件与循环

现代 TypeScript 代码优先使用 const，确实需要重新赋值时再使用 let：

const siteName = "My Blog"
let visits = 0
visits += 1

条件和循环语法与 JavaScript 一致：

if (visits > 0) {
  console.log("Welcome back")
}

for (const name of ["Alice", "Bob"]) {
  console.log(name)
}

五、函数

函数可以标注参数类型和返回值类型：

function add(left: number, right: number): number {
  return left + right
}

可选参数使用 ?，默认参数直接提供默认值：

function greet(name: string, prefix = "Hello", suffix?: string): string {
  return `${prefix}, ${name}${suffix ?? ""}`
}

剩余参数必须使用数组类型：

function join(separator: string, ...items: string[]): string {
  return items.join(separator)
}

箭头函数适合简短的回调：

const doubled = [1, 2, 3].map(value => value * 2)

同一个函数需要支持多组调用方式时，可以使用函数重载；实现签名则负责处理所有情况：

function format(value: string): string
function format(value: number): string
function format(value: string | number): string {
  return String(value)
}

六、数组、元组、对象与 Map

元组用于描述数量和位置确定的元素：

type UserRecord = [id: number, name: string]
const user: UserRecord = [1, "Alice"]

const id = user[0]
const name = user[1]

普通业务对象可以通过类型别名描述：

type Article = {
  id: number
  title: string
  published: boolean
}

Map 适合维护键值映射：

const roles = new Map<number, string>()
roles.set(1, "admin")
roles.set(2, "editor")

roles.get(1)    // string | undefined
roles.has(2)    // true
roles.delete(2)

七、联合类型与类型收窄

联合类型表示一个值可能属于多个类型：

function normalizeId(id: string | number): string {
  return typeof id === "number" ? String(id) : id.trim()
}

这里的 typeof 不只是运行时判断，也会帮助 TypeScript 缩小类型范围。实际项目还可以使用 in、instanceof、自定义类型守卫和判别联合。

type Result =
  | { status: "success"; data: string }
  | { status: "error"; message: string }

function printResult(result: Result): void {
  if (result.status === "success") {
    console.log(result.data)
  } else {
    console.error(result.message)
  }
}

八、接口、类与泛型

1. 接口

接口用于描述对象需要具备的结构：

interface Person {
  readonly id: number
  name: string
  age?: number
}

readonly 表示属性只读，? 表示属性可选。接口还可以继承：

interface Employee extends Person {
  department: string
}

2. 类

class UserService {
  constructor(private readonly prefix: string) {}

  getDisplayName(name: string): string {
    return `${this.prefix}${name}`
  }
}

类适合封装状态与行为。public、protected 和 private 可以控制成员的访问范围。

3. 泛型

泛型让类型关系可以被复用，同时保留调用时的具体类型：

function first<T>(items: T[]): T | undefined {
  return items[0]
}

const firstNumber = first([1, 2, 3])     // number | undefined
const firstName = first(["A", "B"])    // string | undefined

通过约束可以规定泛型至少具备哪些能力：

function getId<T extends { id: number }>(value: T): number {
  return value.id
}

九、模块、命名空间与声明文件

现代 TypeScript 项目主要使用 ES Modules。通过 export 暴露成员，通过 import 引入：

// math.ts
export function add(a: number, b: number): number {
  return a + b
}

// app.ts
import { add } from "./math"
console.log(add(1, 2))

命名空间主要出现在旧项目或不使用模块加载器的特殊场景，新项目通常优先使用 ES Modules。

声明文件以 .d.ts 结尾，用来描述已有 JavaScript 代码的类型。例如为全局变量补充声明：

declare const jQuery: (selector: string) => unknown

许多第三方库已经自带类型；没有内置类型时，也可以查看对应的 @types/* 包。

十、tsconfig.json 与工程化

tsconfig.json 决定 TypeScript 如何检查和生成代码。入门阶段尤其值得关注：

{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ESNext",
    "strict": true,
    "noUncheckedIndexedAccess": true,
    "skipLibCheck": true
  }
}

建议尽早启用 strict，让类型系统真正发挥作用。ESLint 负责代码质量与常见错误检查，Prettier 负责统一格式；二者职责不同，可以配合使用。

十一、推荐学习顺序

可以按下面的顺序推进：

  1. 熟悉 JavaScript 基础与 ES Modules。
  2. 掌握基础类型、数组、对象和函数标注。
  3. 理解类型推断、联合类型与类型收窄。
  4. 学习接口、类型别名、类和泛型。
  5. 掌握 keyof、typeof 和常用工具类型。
  6. 熟悉 tsconfig.json，在真实项目中开启严格模式。
  7. 再进入条件类型、映射类型和类型体操等进阶内容。

总结

学习 TypeScript 的重点不是为所有变量补上类型，而是理解数据之间的约束关系。先用基础类型描述输入和输出，再通过类型收窄、接口和泛型表达更复杂的关系，最后结合严格模式和工程化工具把类型检查融入日常开发。

思维导图适合帮助我们建立全局视角，真正掌握 TypeScript 仍需要把每个知识点放进可运行的小项目中反复练习。
